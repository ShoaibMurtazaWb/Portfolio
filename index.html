SHEET 01 Profile

Two ways of looking at the same problem.

I'm a frontend developer who builds with React and Next.js — component‑based UI, API integration, responsive layouts, the usual craft of shipping a real product. But I came to it through statistics and data science, which means I rarely design a screen without thinking about the structure underneath it: what state it holds, what a user actually needs to see first, how a dashboard should read at a glance.

Day to day that looks like Power BI dashboards and DAX measures on one side, and React components, Tailwind layouts, and MongoDB‑backed apps on the other. Same instinct, two outputs.
